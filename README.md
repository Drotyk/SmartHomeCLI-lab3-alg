 Smart Home CLI SystemКонсольна програма на C++ для керування системою "Розумний будинок". Проект розроблено в рамках курсу "Алгоритмізація та програмування" з акцентом на використання об'єктно-орієнтованого програмування (ООП), патернів проектування та принципів SOLID.
Опис функціоналуСистема дозволяє користувачеві взаємодіяти з віртуальними пристроями через командний рядок.Основні можливості:Додавання пристроїв: Підтримка різних типів (Світло, Кондиціонер).Керування станом: Увімкнення та вимкнення пристроїв.Система Undo (Відкат): Можливість скасувати останню дію (патерн Command).Моніторинг: Перегляд статусу всіх підключених пристроїв.Інтуїтивний інтерфейс: Кольоровий вивід повідомлень та обробка помилок.
 Технології та ПатерниПроект написаний на C++17 з використанням стандартної бібліотеки (STL, smart pointers).
 Реалізовані патерни проектування (Design Patterns):Command (Команда):Усі дії (увімкнення/вимкнення) інкапсульовані в класи TurnOnCommand та TurnOffCommand. Це дозволяє зберігати історію дій та реалізувати функцію undo.Factory Method (Фабричний метод):Клас DeviceFactory відповідає за створення об'єктів пристроїв. Це спрощує додавання нових типів техніки без зміни основного коду програми.Bridge / Abstraction (Абстракція):Використання інтерфейсу IDevice дозволяє системі працювати з будь-якими пристроями однаково, не знаючи їх конкретної реалізації.Відповідність принципам SOLID:S (SRP) — Принцип єдиної відповідальності:DeviceManager — лише зберігає пристрої.CommandInvoker — лише керує виконанням та історією команд.SmartHomeCLI — відповідає лише за введення/виведення даних.O (OCP) — Принцип відкритості/закритості:Система відкрита для розширення (можна додати новий клас Heater, наслідувавши IDevice), але закрита для модифікації (не треба змінювати логіку SmartHomeCLI).L (LSP) — Принцип підстановки Лісков:Будь-який клас-спадкоємець IDevice (наприклад, Light) коректно замінює базовий тип.D (DIP) — Принцип інверсії залежностей:Команди залежать від абстракції IDevice, а не від конкретних класів Light чи AC. Компіляція (Windows)Можна використовувати Visual Studio або MinGW:DOSg++ -std=c++17 main.cpp -o smarthome.exe
